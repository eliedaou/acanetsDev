5. Known Issues:
A. HeteroSpark's performance will decrease dramatically when using as Remote GPU mode because of the RMI overhead.
B. Algorithms with huge parameter transferring between CPU and GPU is known to have bad performance. We are still working on parameter tuning on these algorithms.

6. Current status:
1. Rewriting RMI communication protocol by using Spring framework.
2. Developing JNI wrapper automator by using Swig.
3. Working on a list of machine learning algorithms.
